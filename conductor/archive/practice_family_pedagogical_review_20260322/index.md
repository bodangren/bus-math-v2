# Track: Practice Family Pedagogical Review

Audit every practice engine family for pedagogical effectiveness and multi-stage versatility: can each family convincingly serve teacher demonstration, guided/scaffolded practice, and authentic assessment — each with unique, deterministically generated values?

## Artifacts

- **Specification**: [./spec.md](./spec.md)

## Summary

This track is a discussion-driven review, not an implementation track. For each practice family (A through P and beyond), we evaluate:

1. **Pedagogical soundness** — Does the family target a clear learning objective? Does the activity type (categorization, matrix selection, journal entry, etc.) match the cognitive demand of that objective?
2. **Multi-stage generation** — Can the engine produce distinct problem sets for three classroom stages:
   - **Teacher demonstration** — a curated, predictable set the instructor can walk through
   - **Guided practice** — a scaffolded set with hints and feedback, unique from the demo
   - **Authentic assessment** — a fresh set with no scaffolding, unique again
3. **Variety and replayability** — Are there enough parameter axes (scenario keys, account pools, archetypes, seed values) to avoid repetition across sections and semesters?
4. **Misconception coverage** — Does the grading/feedback layer surface the right misconception tags for the objective?
5. **Gaps and recommendations** — What's missing or weak? What new families or parameter axes would strengthen the curriculum?
